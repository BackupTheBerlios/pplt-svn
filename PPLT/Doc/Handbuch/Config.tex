	Obwohl es für eine Bibliothek ungewöhnlich ist, eine Konfiguration zu besitzen, lassen sich
	bei der PPLT in guter Posix-Manier, einige Einstellungen mittels einer Konfigurationsdatei tätigen.
	
	Die Eistellmöglichkeiten sind aber eher bescheiden, so dass dieses Kapitel eher kurz werden
	dürfte. Eigentlich lassen sich nur ein paar Verszeichnisse, in denen das System
	nach Modulen und Icons suchen soll, die Sprache und die so genannten LogLevel
	einstellen.
	
	Wie schon oben erwähnt, besitzt die PPLT eine Konfigurationsdatei. Diese kann mit einem
	Texteditor manipuliert werden. Dem routinierten Windowsbenutzer dürfte diese Art der 
	Konfiguration ungewohnt erscheinen. Wo hingegen Anwender mit Linux-Erfahrung darüber
	müde lächeln. 
	
	Die Konfigurationsdatei befindet sich in dem Verzeichnis, in dem die PPLT all ihre Daten,
	wie die Icons für das PPLT-Center oder eben die Module ablegt. Dieser Ordner heißt 
	\texttt{PPLT} und liegt unter Windows in dem Verzeichnis, in das sie Python installiert 
	haben. Unter Linux sollten sie einmal im Verzeichnis \texttt{/usr} oder \texttt{/usr/local}
	nachsehen. Die Datei heißt \texttt{PPLT.conf}. Öffnen sie diese mit dem Texteditor ihrer wahl.
	
	Der Aufbau der Konfiguration ist recht einfach. In eckigen Klammern stehen die einzelnen
	Bereiche der Konfiguration. In der PPLT.conf wären das der Bereich \texttt{lang}
	in dem alle Einstellungen der Sprache betreffend gemacht werden können. Im Abschnitt
	\texttt{logging} können sie das Meldungsverhalten der PPLT beeinflussen. Im letzten
	Abschnitt \texttt{path} werden die Pfade für PPLT, wie zum Beispiel der Pfad zu den 
	Icons gesetzt.
	
	\section{Die Spache}
	In diesem Bereich können sie die Sprache der PPLT und vor allem die Spache des PPLT-Centers
	festlegen. Auffällig ist, dass hier zwei Spacheinträge vohanden sind. Der erste Eintrag
	beschreibt die primäre Sprache. Wenn sie also eine deutsche Übersetzung wünschen,
	sollten sie bei \texttt{language:} \texttt{de} eintragen. 
	
	Der Eintrag \texttt{alt-lang} bedeutet soviel wie "`Alternativ Sprache"' dieser Eintrag
	wird dazu benutzt eine Sprache zu definieren, auf die zurückgereiffen werden soll, wenn
	die primäre Sprache nicht verfügbar ist. Deshalb sollten sie hier den Eintrag auf
	\texttt{en} belassen.
	
	Eine sinnvolle Sprachkonfiguration für die Sprache Deutsch wäre:
	\begin{verbatim}
[lang]
language: de
alt-lang: en
	\end{verbatim}
	
	
	\section{Das Logging}
	Als logging bezeichnet man Verarbeitung und Ausgabe von Nachrichten, die wärend
	des Prgrammablaufes auftreten. Solchen Nachrichten haben verschiedene Gewichtungen
	und können von einfachen informativen Meldungen bis hin zu Fehlermedungen 
	klassifiziert werden. In diesem Abschnitt können sie festlegen welche Gewichtung 
	einen Meldung haben muss um überhaupt verarbeitet zu werden. Meldungen 
	geringerer Gewichtung werden ignoriert. Des weiteren können sie festlegen, wie
	diese gefilterten Meldungen ausgegeben werden sollen. 
	
	\subsection{LogLevel}
	Die PPLT klassifiziert die Meldungen in \texttt{debug}, \texttt{info},
	\texttt{warning}, \texttt{error} und \texttt{fatal}. Wobei \texttt{debug}
	die niedrigste und \texttt{fatal} die höchste Stufe darstellt. 
	
	Als \texttt{debug} werden all die Meldungen deklariert, die zur Fehlerfindung im
	Program hilfreich seien könnten. Dies sind aber sehr viele. So dass, wenn sie dieses
	Level auswählen, sie sehr viele Meldungen erhalten. Daher ist diese Gewichtung
	für den normalen Betrieb eher ungeeignet.
	
	Als \texttt{info} werden informative Medungen deklariert, die durchaus für den Anwender 
	interresant sind. Es ist also sinnvoll diesen Filter für die Konfiguration zu verwenden.
	Setzten sie daher bitte in der Konfiguration:
	\begin{verbatim}
[logging]
CoreLevel: info
PPLTLevel: info
File: No
SysLog: No
	\end{verbatim}
	Damit werden Nachrichten mit der Gewichtung \texttt{info} oder höher verarbeitet.
	
	Meldungen, mit der Gewichtung \texttt{warning}, stellen Warnungen dar. Diese Meldungen
	weisen gegenbenenfalls auf Fehlfunktionen hin. Jedoch deuten diese nich zwingend auf
	eine Fehlfunktion hin.
	
	Mit \texttt{error} oder \texttt{fatal} werden Nachrichten gewichtet, die auf einen
	Fehler hinwiesen. Jedoch bedeutet \texttt{fatal}, dass dieser Fehler aus einem Fehler
	im Programm und nich auf einer Fehlbedienung herrüht. Tritt also ein Fehler mit der
	Gewichtung \texttt{fatal} auf, wenden sie sich bitte an den Autor.
	
	In der Konfiguartionsdatei finden sie zwei Level- oder Filter-Ein\-träge. Der 
	Eintrag \texttt{CoreLevel} setzt den Filter für die Kernbibliothek. Der Filter
	für die PPLT-Bibliothek und das PPLT-Center kann mit dem Eintrag 
	\texttt{PPLTLevel} gesetzt werden. Es ist durchaus sinnvoll für beide Einträge 
	die selben Filter zu verwenden.
	
	\subsection{Ausgabeumleitung}
	Es ist möglich die Ausgabe der gefilterten Meldungen umzuleiten. Zum Beispiel in eine
	Datei oder zu einem Logging-System des Betriebsystems\footnote{Steht bis jetzt nur für 
	Posix-Systeme zur Verfügung}. Dazu existieren die Eiträge \texttt{File} sowie
	\texttt{Syslog} in der Konfiguration. Wenn sie beide Einträge auf \texttt{No} belassen,
	werden die Nachrichten auf der \texttt{StdErr} ausgegeben. 
	
	Wenn sie jedoch bei \texttt{File} den Pfad einer Datei angeben, werden die Meldungen
	in eben dieser Datei abgelegt. 
	
	Wenn sie \texttt{SysLog} auf \texttt{Yes} setzten, werden die gefilterten Nachrichten
	an den SysLog-Dämon des Betriebssystems\footnote{Existiert nur unter Posix Systemen.}
	gesendet. Diese Einstellung überschreibt den \texttt{File}-Eintrag. Das bedeutet,
	wenn sie \texttt{SysLog} auf \texttt{Yes} gesetzt haben und eine Datei bei \texttt{File}
	angegeben haben, werden jedoch alle Meldungen ausschließlich an den SysLog-Dämon 
	gesendet.
	
	Eine sinnvolle Konfiguration wäre:
	\begin{verbatim}
[logging]
CoreLevel: info
PPLTLevel: info

File: No
SysLog: No
	\end{verbatim}
	
		
	\section{Pfadeinstellungen}
	\textbf{Achtung:} Änderungen an den Pfadeinstellungen sollten nur von erfahrenen Benutztern
	gemacht werden. Fehleinstellungen können das Verhalten des Programms starkt beeinflussen.
	
	Im Abschnitt \texttt{path} werden sie Pfade zu den Dateien, die die PPLT benötigt gesetzt.
	Dazu ge\-hören die Module, Ge\-räte, Server, Icons und die Benutzer\-daten\-bank-Datei. 
	
	Für alle Einträge ist \texttt{AUTOMATIC} eine gültige Option. Damit sagen sie dem System,
	dass es die Standard-Pfade benutzen soll. 
	
	Mit \texttt{BasePath} wird der Pfad zu dem PPLT Verzeichnis, in dem sich auch die 
	Konfigurationsdatei befindet, gesetzt. Dem Eintrag \texttt{UserDB} übergeben sie 
	die Benutzterdatenbank-Datei, somit ist es möglich verschiedene Datenbanken für
	die PPLT zu unterhalten. Der Eintrag \texttt{DBPath} ist obsolet und setzt den Pfad auf
	das Verzeichnis, indem sich die Server- und Geräte-Dateien befinden. Mit 
	\texttt{IconPath} wird der Pfad auf das Verzeichnis gesetzt, welches die Icons
	für das PPLT-Center beinhaltet.
	
	Bitte ändern sie die Einträge nur, wenn sie wissen was sie tun.
