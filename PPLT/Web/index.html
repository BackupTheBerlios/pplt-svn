::Intro

In this page I will give you a (realy) short intoduction into the PPLT concept, if you want ot known more, please follow the links in this text.

Main Goals
The main goal was to have a flexible, modular and platform-independent system, that grap information from many different systems over differen interfaces meaning controlers and intelligent sensors and manage them together in one, so called, Symbol-Tree.

The second goal was to have a own user and group management to control the access to this central Symbol-Tree.

At least it should be able to export this symbols to other sysmtems by modules. So it is not bound to a single visualiziation.

Disign
You can split the PPLT system into 3 parts.

One part imports information and data from systems for example from controlers or sensors. This part is a little bit difficault to use, bacause you have to handle with a lot of modules to get information from a controler. You need for example a module for the interface, the BUS protocol and one for the command-messageformat to read out values of a markers in a controller. But this is a main goal of this project because you only need to replace the interface and protocol modules to talk to the same controller over an other BUS-system. This makes this system very flexible.

The other part manage this imported information into a file-system like Symbol-Tree. Imagine this Tree as a small file-system with folders and files. Each file (called symbol) represent for example a marker into a controller or the actual value of a sensor. To manage them better, if you have many of them, there are folders. This Symbol-Tree have also (like file-systems) a right-managment. This means you can define permissions to a spezific symbol or folder for other users and groups.

The last part exports this symboltree to other systems, like visualisations or databases. This export-part also work with modules, so it is possible to process the information in the symbol-tree in a many applications (if there are implemented with a module). For example there is a module which use XML-RPC to export this symbols for other applications.

But the best of all is, it was written in Python.
Supported (Implemented) Hardware
I will give you a list of Hardware, mostly Controler, i tested with PPLT. It may be possible that more will work because it hardly depends on the constellation of the used modules.
  • NAIS FP0/FP2 - I tested them over the ToolPort. It was also possible to access them over the tunneled ToolPort of an NAIS Web-Server.
  • NAIS A200 - This is an industrial image-checker. To access it i used the build in RS232 port.
  • Siemens SIMATIC S7-200 - I used the PPI-Cable in FreePort mode.
  • Siemens S55 mobile phone. I am sure that other GSM compatible mobile phones will also work fine.
