Ziele
Das Hauptziel was es, ein flexibles, modulares und vor allem platformunabhängiges System zu schaffen, das auf Informationen aus verschiedenen Quellen über unterschiedlichste Schnittstellen zu greifen kann. Dabei vor allem auf Steuerungen und sog. intelligente Sensoren. All diese Informationen sollen dann zentral in einem sog. Symbol-Baum abgelegt werden.

Als zweites sollte das System eine eigene Benutzer- und Gruppenverwaltung haben um den Zugriff auf den Symbol-Baum regeln zu können.

Als letztes sollte es die Möglichkeit geben, über Module diesen Symbolbaum wieder anderen Systemen zur Verfügung zu stellen. So dass man nicht an eine bestimmte Applikation, wie zum Beispiel eine Visualisierung, gebunden ist.

Disign
Im Groben lässt sich das PPLT System in drei Bereiche teilen.

Der erste Teil ist etwas schwierig zu verstehen, weil sie mit einer Vielzahl an Modulen arbeiten müssen. Er ist dazu da, Informationen und Daten aus anderen Systemen zu importieren. Dazu benötigt man zum Beispiel, um aus den Merkern einer Steuerung zu lesen, ein Modul für die Schnittstelle, eins für das BUS-Protokoll und eins das das Kommando-Nachrichtenformat der Steuerung implementiert, bevor man die Daten auslesen kann. So müssen sie zu mindest wissen, welches BUS-Protokoll, und welche Schnittstelle verwendet wird um mit der Steuerung zu kommunizieren. Dies ist jedoch auch einer Vorteile des PPLT Systems, da sie nun lediglich das Schnittstellen- und das BUS-Modul austauschen müssen um mit der Steuerung über einen anderen BUS zu kommunizieren.

Der andere Teil verwaltet diese importierten Informationen in einem Dateisystem ähnlichen Symbol-Baum. Stellen sie sich diesen Baum als eine Art mini Dateisystem mit Verzeichnissen und Dateien vor. Jede Datei (hier Symbol genannt) representiert zum Beispiel einen Merker in einer Steuerung oder den aktuellen Wert eines Sensors. Um diese ganzen Symbole besser organisieren zu können, gibt es Verzeichnisse oder Ordner.
Wie jedes Datei-System besitzt auch der Symbol-Baum eine Rechteverwaltung, mit der sie den Zugriff von Benutzern oder Gruppen auf jedes Symbol oder jedes Verzeichnis einzeln regeln.

Der lezte Teil exportiert diesen Symbol-Baum zu anderen Systemen, wie zum Beispiel Visualisierungen oder Datenbanken. Dieser Export-Teil arbeitet auch mit Modulen, so dass sie die Daten im Symbolbaum in belibiegen Anwendungen verwenden können (insofern für diese ein Export-Modul existiert). Es gibt ein Export-Modul, das die XML-RPC Bibliothek verwendet um den Symbol-Baum zu exportieren. Auf diese Weise können sie auf die Symbole in ihren selbgeschriebenen Applikationenen zugreifen.

Das Beste ist jedoch, dass es in Python geschrieben ist.
Unterstützte Hardware
Hier habe ich die Geräte, meißt Steuerungen, aufgelistet, für die es Module für PPLT gibt. Es ist möglich dass noch andere Geräte mit diesen Modulen angesprochen werden können, da sehr von der Art und Weise abhängt, wie die Module aneinander gekoppelt werden.
  • NAIS FP0/FP2 - Es ist möglich die Merker dieser beiden Steuerungen über den ToolPort anzusprechen. Es ist ebenfalls möglich die FP0 (wenn vorhanden) über den Web Server FP-WEB zu erreichen.
  • NAIS A200 - Das ist ein sog. Image-Checker zu optischen Qualitätsprüfung. Um ihn anzusprechen nutzte ich die eingebaute RS232 Schnittstelle.
  • Siemens SIMATIC S7-200 - PPLT kann auf die Merker dieser Steuerung über den PPI-Bus zugreifen.
  • Siemens S55 Handy. Ich bin mir sicher, dass auch andere Handys mit diesem Modul ausgelesen werden können, die dem GSM Standard entsprechen.
